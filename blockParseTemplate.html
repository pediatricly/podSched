How does El Nino work?

El Nino reads the messy schedule data on the Amion website and turns it into something only a robot could love - beautifully elaborate, machine-readable data on the entire residency program's schedule. This has some great uses:

  • lunchMoney
    • Tally up how much cash to put on meal cards based on their rotation week-by-week.
  • confAttend
    • Get the "denominator" for conference attendence. Count up how many conferences residents should have attended based on their block schedules.
  • podSched
    • Getting multiple residents on different schedules in the same place at the same time is a pain!
    • podSched goes through an arbitrary subset of residents' schedules and outputs a ranked, scored list of dates when they are on the most aligned and convenient rotations.
    • For example, if you want to schedule an evening meeting, you can score for shifts that end around 5pm to pick an evening when everyone is available. (Hence the name - this was developed to schedule PLUS pod dinners.)

BUT, to do any of this, you need to make sure El Nino is working with an updated schedule. We all know the Amion schedule changes frequently. And that's why you are here.

This should have updated these main output files:

  • $outfile
    • NB for superusers: These links are actually to other Python CGI scripts that just prints the data file. You can't directly download the file for security reasons.
    • Contains the main data, all resident schedules and block dates, as a Python file.
    • You'll probably never look at this but nice to know it's there. (And to have it for debugging.)
  • $CoC
    • Simple spreadsheet (csv) of all the residents' Amion names with their continuity clinic day as listed in Amion. (NB: Will have weird/missing output for chiefs and super-seniors.)
    • Very useful for the schedule template spreadsheets.
  • $allRots
    • Even simpler csv, just a list of all the rotations in this year's block schedule across all years.
    • Very useful for the other El Nino functions where you'll need scores or expected values for each rotation.

Here are some summary stats for sanity checking. El Nino scraped data on:

  • $totalRots rotations in total. Typical is very roughly 1500.
  • $totRes residents in total. Typical is about 90.
    • R3s: $R3str
    • R2s: $R2str
    • R1s: $R1str
  • Your new date is $updated (yyyy-mm-dd).
  • Your block 1 split dates are:
    • $block1split1 for R1s
    • $block1split23 for R2s and R3s
