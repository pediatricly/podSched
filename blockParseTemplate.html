Congratulations! You have just updated El Nino's database!

This should have updated these main output files:

  • $outfile
    • Contains the main data, all resident schedules and block dates, as a Python file.
    • You'll probably never look at this but nice to know it's there. (And to have it for debugging.)
  • $CoC
    • Simple spreadsheet (csv) of all the residents' Amion names with their continuity clinic day as listed in Amion. (NB: Will have weird/missing output for chiefs and super-seniors.)
    • Very useful for the schedule template spreadsheets.
    • After it loads in your browser, Right Click > Save As... > name it a .csv file and you can open it easily in Excel.
  • $allRots
    • Even simpler csv, just a list of all the rotations in this year's block schedule across all years.
    • Very useful for the other El Nino functions where you'll need scores or expected values for each rotation.
    • Same deal, Right Click > Save As... > name it a .csv file and you can open it easily in Excel.
  • NB: These links are actually to other Python CGI scripts that just prints the data file. You can't directly download the files for security reasons.

Here are some summary stats for sanity checking. El Nino scraped data on:

  • $totalRots rotations in total. Typical is very roughly 1500.
  • $totRes residents in total. Typical is about 90.
    • R3s: $R3str
    • R2s: $R2str
    • R1s: $R1str
  • Your new update date is $updated (yyyy-mm-dd).
  • Your block 1 split dates are:
    • $block1split1 for R1s (yyyy-mm-dd)
    • $block1split23 for R2s and R3s (yyyy-mm-dd)
