Any residents with missing data are printed in the table below.

  • This is normal for super-seniors (+ chiefs), but they should be filtered out of this list.
    • Tell Mike if super-seniors are in the table.
  • If normal-schedule residents are listed here:
    • Most likely, they're on a new rotation. (Or an old rotation with a new name in Amion - to El Nino, "Purple1" and "purple 1" are completely different rotations.):
      • You'll need to define the weekly number of AM and noon conferences expected for this new rotation. Email those to Mike.
    • But, if that doesn't fix it, there may be a problem with how El Nino is reading their schedule.
      • These often show up as names without block numbers or rotation names because El Nino couldn't figure out either.
      • El Nino can only read full-week rotations. Rarely, residents may have shorter rotations esp in Block 1 or 13. Double check any of these manually.
      • If none of those explain it, tell Mike.
