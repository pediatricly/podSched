What's the deal with this list of allotments by rotation?

This is a simple spreadsheet (csv or comma-separated file) uploaded by a chief or admin that El Nino uses to figure out how much money residents should receive based on their rotation. It looks like this:

You need to follow that format exactly for it to work. Easiest is to modify the current list, but you can also follow these criteria:

  1. It must be saved as a .csv file. (Excel etc can save as this format.)
  2. It must have only 2 columns:
    1. Rotation
    2. cashWk
  3. The rotation names in column A need to match those in Amion exactly. You can get the latest list in Amion from El Nino (based on the last El Nino update).
  4. The cash amounts in column B must be weekly and must be numbers only.

How does El Nino work?

El Nino reads the messy schedule data on the Amion website and turns it into something only a robot could love - beautifully elaborate, machine-readable data on the entire residency program's schedule. This has some great uses:

  • Lunch Money
    • Tally up how much cash to put on meal cards based on their rotation week-by-week.
  • confAttend - not available online yet
    • Get the "denominator" for conference attendence. Count up how many conferences residents should have attended based on their block schedules.
  • Meeting Planner
    • Getting multiple residents on different schedules in the same place at the same time is a pain!
    • Meeting Planner goes through an arbitrary subset of residents' schedules and outputs a ranked, scored list of dates when they are on the most aligned and convenient rotations.
    • For example, if you want to schedule an evening meeting, you can score for shifts that end around 5pm to pick an evening when everyone is available. (Hence the original name "podSched" - this was developed to schedule PLUS pod dinners.)

BUT, to do any of this, you need to make sure El Nino is working with an updated schedule. We all know the Amion schedule changes frequently. And that's why it all starts with Updater.
