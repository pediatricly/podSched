El Nino has just calculated all the resident meal card allotments you requested.

But first, check for 2 errors!

Error - Residents with Missing Rotation Data

  • This is normal for super-seniors (which often include chiefs). They will always be on this list.
  • But if normal-schedule resident are listed here:
    • Most likely, they're on a new rotation - see below.
    • But, if that doesn't fix it, there may be a problem with how El Nino is reading their schedule.
      • El Nino can only read full-week rotations. Rarely, residents may have shorter rotations esp in Block 1 or 13. Double check any of these manually.
      • If none of those explain it, tell Mike.
$errResStr

Error - Rotations Missing Money Data

  • New / unusual rotations may show up here because El Nino does not know how much money they should be assigned.
  • This is common and easy to fix - go back to the entry form and upload a new version of $rotCashcsv with the new rotations in new rows at the bottom.
    • Remember, these must be entered in the csv exactly the way they are spelled here. Spacing, capitalization and punctuation must all match perfectly. (To El Nino, "Purple1" and "purple 1" are completely different rotations.)
$errRotStr

This should have updated these main output files:

  • $csvOutFile
    • Contains the main data, allotments resident by block, in csv format.
    • After it loads in your browser, Right Click > Save As... > change the extension to .csv file and you can open it easily in Excel.
  • $rotCashcsv
    • View the current csv of meal cash allotments by rotation that El Nino used.
  • $errFile
    • Simple text output of errors this program encountered. Such errors occur when a resident has blanks in their schedule data, e.g., super-seniors have a bunch of blank blocks after they graduated. This file logs them for debugging.
