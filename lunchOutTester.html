How does El Nino work?

El Nino reads the messy schedule data on the Amion website and turns it into something only a robot could love - beautifully elaborate, machine-readable data on the entire residency program's schedule. This has some great uses:

  • Lunch Money
    • Tally up how much cash to put on meal cards based on their rotation week-by-week.
  • confAttend - not available online yet
    • Get the "denominator" for conference attendence. Count up how many conferences residents should have attended based on their block schedules.
  • Meeting Planner
    • Getting multiple residents on different schedules in the same place at the same time is a pain!
    • Meeting Planner goes through an arbitrary subset of residents' schedules and outputs a ranked, scored list of dates when they are on the most aligned and convenient rotations.
    • For example, if you want to schedule an evening meeting, you can score for shifts that end around 5pm to pick an evening when everyone is available. (Hence the name - this was developed to schedule PLUS pod dinners.)

BUT, to do any of this, you need to make sure El Nino is working with an updated schedule. We all know the Amion schedule changes frequently. And that's why it all starts with Updater.

Error - Residents with Missing Rotation Data

  • This is normal for super-seniors (which often include chiefs). They will always be on this list.
  • But if normal-schedule resident are listed here:
    • Most likely, they're on a new rotation - see below.
    • But, if that doesn't fix it, there may be a problem with how El Nino is reading their schedule.
      • El Nino can only read full-week rotations. Rarely, residents may have shorter rotations esp in Block 1 or 13. Double check any of these manually.
      • If none of those explain it, tell Mike.

Error - Rotations Missing Money Data

  • New / unusual rotations may show up here because El Nino does not know how much money they should be assigned.
  • This is common and easy to fix - go back to the entry form and upload a new version of foodCashByWeek.csv with the new rotations.
    • Remember, these must be entered in the csv exactly the way they are spelled here. (To El Nino, "Purple1" and "Purple 1" are 2 completely different rotations.)

This should have updated these main output files:

  • lunchMoneyOut.csv
    • Contains the main data, allotments resident by block, in csv format.
    • After it loads in your browser, Right Click > Save As... > name it a .csv file and you can open it easily in Excel.
  • foodCashByWeek.csv
    • View the current csv of meal cash allotments by rotation that El Nino used.
  • lunchMoneyErrors.txt
    • Simple text output of errors this program encounter. Such errors occur when a resident has blanks in their schedule data, e.g., super-seniors have a bunch of blank blocks after they graduated. This file logs them for debugging.
