This is where you can update El Nino's database

Updates are big deal:

  • El Nino reads the schedule from Amion and saves its own data.
  • El Nino uses those data for its various functions.
  • Of course, Amion changes all the time...
  • So it's important to update El Nino before using it for critical tasks (e.g., assigning lunch money).

El Nino was last updated on:

  • $updated (yyyy-mm-dd)

Block 1 Split Dates

Please let the Chiefs ONLY do this section.
It only needs to be used once a year in July but will break the system if done wrong.

El Nino can figure out most of the block schedule, but it needs you to set the "split dates" for block 1. You should only have to set this manually once a year in July.
Split dates are the Mondays when second-half blocks (eg the SFX in "JEOP | SFX") start. Because block 1 rarely starts on a Monday, el Nino needs you to tell it when to split block 1 (for both the R1 and R2/3 schedules).

Block 1 Split, R1: Works best in Google Chrome. If your browser is showing a plain text entry box, you must enter the date as "YYYY-MM-DD". Currently set to $block1split1
Block 1 Split, R2/3s: Works best in Google Chrome. If your browser is showing a plain text entry box, you must enter the date as "YYYY-MM-DD". Currently set to $block1split23

If those current dates are still accurate and/or you are not a Chief Resident, leave those boxes blank and simply hit Submit to update the database.
